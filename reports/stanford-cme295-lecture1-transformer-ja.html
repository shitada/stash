    2. Transformer へ至る問題解決の連鎖

    講義の構成は、前の手法の弱点が次の手法の動機になるよう設計されている。単語を数値へ変えるだけでは語順と文脈が消え、RNN で語順を取り戻すと長距離依存と逐次計算が問題になり、Attention で過去の必要箇所へ直接つなぎ、Transformer では Self-attention を中心に再構成する。

    flowchart TD A["1. 文字列を token へ
    Tokenization"] --> B["2. token の意味をベクトルへ
    Word2vec"] B --> C["3. 語順と逐次文脈を保持
    RNN / LSTM"] C --> D["4. 必要箇所を直接参照
    Attention → Self-attention → Transformer"]

    図をクリックすると拡大・ドラッグ・ズームできます。

    まずは3用語をひとことで
    • Word2vec：単語を数値ベクトルに変え、似た文脈で使われる単語ほど近くに配置する手法。単語ごとに固定の「意味の座標」を作るイメージ。
    • RNN（Recurrent Neural Network）：文章を先頭から順に読み、直前までの内容を短いメモとして次の単語へ渡すニューラルネットワーク。
    • LSTM（Long Short-Term Memory）：RNN のメモに「覚える・忘れる・読み出す」を調整する仕組みを加え、前に出た情報をより長く残しやすくした改良版。

    大まかには、Word2vec が単語の意味を数値化し、RNN がその単語を順番に読み、LSTM が RNN の記憶を長持ちさせる、という関係にある。

    歴史的にも、RNN 系の考えは1980年代、LSTM は1990年代、Word2vec は2013年、ニューラル機械翻訳の Attention は2014年、Transformer は2017年に登場した。講師は、古いアイデアが当時すぐ現在の規模へ進めなかった背景として、インターネット由来のデータと計算資源の不足を挙げる。2020年代にはデータ・計算量・モデル規模が拡大し、LLM が広く認知されるようになった。

    3. NLP タスクと評価指標

    NLP（Natural Language Processing）は、自然言語テキストを計算機で扱う分野である。講義では、出力の形を基準にタスクを3群へ整理する。

    講義での区分入出力代表例評価の考え方
    Classificationテキスト → 1ラベル感情分析、意図検出、言語検出、トピック分類Accuracy、Precision、Recall、F1
    Multi-classificationテキスト → 複数位置のラベルNER、品詞タグ付け、依存・構文解析トークン単位またはエンティティ種別単位で集計
    Generationテキスト → 可変長テキスト機械翻訳、質問応答、要約、コード・詩の生成BLEU、ROUGE、Perplexity など
    用語上の注意

    講義の “multi-classification” は、NER や品詞タグ付けのように複数位置を分類する一群を便宜的にまとめた呼称である。一般には sequence labeling / token classification など、より具体的な名称を使うことが多い。

    分類指標

    • Accuracy：全例のうち正解した割合。99:1 のような不均衡データでは、常に多数派を予測しても高くなり、誤解を招く。
    • Precision：正例と予測したもののうち、実際に正例だった割合。
    • Recall：実際の正例のうち、正例として見つけられた割合。
    • F1：Precision と Recall の調和平均。両者を1つの値で捉える。

    補足：Precision・Recall・F1 の計算と現在の使われ方

    まず「正例」を何とするかを決め、予測と正解の組み合わせを次の4種類に数える。F1 の計算には TP・FP・FN を使い、TN は直接は使わない。

    予測：正例予測：負例
    実際：正例TP（正しく検出）FN（見逃し）
    実際：負例FP（誤検出）TN（正しく除外）
    Precision = TP / (TP + FP)
    Recall = TP / (TP + FN)
    F1 = 2 × (Precision × Recall) / (Precision + Recall) F1 は 2TP / (2TP + FP + FN) とも書ける。通常は 0〜1 で、1に近いほど良い。

    計算例：100件を正例と予測し、そのうち80件が正解なら TP=80、FP=20。実際の正例が120件なら、残り40件を見逃しているので FN=40 となる。Precision は 80 / 100 = 0.80、Recall は 80 / 120 ≈ 0.67、F1 は 2 × 0.80 × 0.67 / (0.80 + 0.67) ≈ 0.73 である。調和平均を使うため、Precision と Recall の片方だけが高くても F1 は高くなりにくい。

    現在の主な使われ方：F1 は、正例が少なく Accuracy だけでは性能を判断しにくい分類や、誤検出と見逃しの両方を重視する場面で使われる。現在の機械学習・NLP ライブラリでも標準的な分類指標として実装されている。

    利用場面使い方注意点
    不均衡な二値分類スパム・不正・異常など、注目する正例クラスの Precision と Recall を1値にまとめる。TN を直接評価しないため、負例側の性能も重要なら混同行列などを併記する。
    多クラス・マルチラベル分類クラスごとの F1 と、micro・macro・weighted などの平均値を報告する。平均方法で値の意味が変わるため、単に「F1」とだけ書かず集計方法を明記する。
    NLP の NER / token classification人名・地名などの抽出結果について Precision・Recall・F1 を測る。Hugging Face の現行ガイドでも seqeval による3指標を使用する。token 単位か、ラベルと範囲が一致した entity 単位かを明記する。
    モデル選択・閾値調整検証データ上の F1 でモデルや判定閾値を比較する。テストデータで閾値を調整しない。業務上の誤検出・見逃しコストが同等でない場合は、Precision、Recall、Fβ、PR曲線なども使う。

    多クラス分類では、各クラスを1対その他の二値問題として評価してから集計する。macro F1 はクラスごとの F1 を同じ重みで平均するため少数クラスの弱さが表れやすい。micro F1 は全クラスの TP・FP・FN を合計してから計算し、各予測を同じ重みで扱う。weighted F1 は各クラスの正解件数（support）で重み付けするため、全体傾向を表しやすい一方で多数クラスの影響が強い。単一ラベルの多クラス分類では micro F1 が Accuracy と一致する点にも注意が必要である。

    F1 は万能な品質指標ではない

    F1 は Precision と Recall を同じ重みで扱う。見逃しの損失が特に大きいなら Recall や F2、誤検出の損失が大きいなら Precision や F0.5 を重視する。また、予測確率の確からしさや生成文の意味的品質は F1 だけでは測れないため、目的に合う複数の指標と併用する。

    生成指標

    機械翻訳では、同じ意味に複数の正しい訳があり得る。WMT のような対訳データセットを使い、候補訳を参照訳と比べる BLEU や ROUGE が使われてきた。ただし、参照文の作成には人手・時間・費用がかかる。講師は、このコースの後半で LLM を使った reference-free 評価へ進むと予告する。

    指標何を見るか向き注意
    BLEU候補訳と参照訳の n-gram 一致を、適合率と短さへのペナルティで評価高いほど良い元論文は主にコーパス単位の人手評価との相関を狙う
    ROUGE参照要約との n-gram や最長共通部分列などの重なり高いほど良い複数方式からなる指標群
    Perplexity正解系列へモデルが割り当てた確率から「驚き」を測る低いほど良い人間の品質判断や下流タスク性能と必ず一致するわけではない

    4. Tokenization：文字列をモデルの処理単位へ

    モデルは文字列をそのまま理解せず、数値を処理する。そこで文章を任意の単位へ分割し、各単位を token と呼ぶ。どの粒度を選ぶかは、語の意味を再利用できるか、未知語に耐えられるか、系列がどれだけ長くなるかのトレードオフになる。

    粒度利点弱点
    Word-level単純で解釈しやすく、系列が比較的短いbear と bears を別物として覚える。語彙が膨らみ、OOV が増える。
    Subword-level接頭辞・接尾辞や共通部分を再利用し、OOV を減らしやすい単語より系列が長くなり、処理量が増える。
    Character-level綴り間違いや大小文字の揺れに強く、未知語がほぼない系列が非常に長く、1文字の埋め込みを意味的に解釈しにくい。
    一次資料による補足：BPE と rare word

    Sennrich ら（ACL 2016）は、頻出する文字列単位を学習して語を subword に分け、希少語や未観測語を既知部分から構成する方法をニューラル機械翻訳へ導入した。これは語彙を有限に保ちながら open vocabulary に近づける考え方である。BPE と WordPiece は同じ subword 系でも、学習規則は同一ではない。

    系列長は後段の計算量へ直結する。細かく切れば未知語耐性は上がる一方、モデルが処理する token 数が増え、学習・推論の時間とコストが増える。講義の質疑では、単一言語なら数万、現在の多言語・コード対応モデルでは数十万規模の語彙もあるという大まかな目安が示されたが、最適値は言語と用途に依存する。

    5. One-hot から Word2vec へ

    One-hot の限界

    語彙サイズを V とすると、One-hot encoding は各 token を長さ V のベクトルで表し、該当位置だけを1にする。識別には使えるが、異なる token 同士の内積は常に0である。したがって、teddy bear と soft のような関連も、無関係な語との違いも表せない。

    講義ではベクトルの向きを比べる cosine similarity を導入する。同方向なら1に近く、直交なら0、反対方向なら−1に近い。ただし講師が質疑で強調したように、これは意味的類似度を測る便利な尺度であって唯一の完全な尺度ではない。

    cosine(x, y) = (x · y) / (||x|| ||y||) 大きさを正規化し、2ベクトルが作る角度に注目する。

    Word2vec の proxy task

    Word2vec は、最終目的である「意味のある埋め込み」を直接教師付けする代わりに、文章中の語を予測する proxy task を解かせる。文脈から語を当てられるなら、モデル内部には言語の規則性が必要になる。その過程で得た中間層の重みを単語ベクトルとして使う。

    方式入力予測
    CBOW対象語の周辺にある複数の語中央の対象語
    Skip-gram中央の対象語周辺に現れる語

    講義の簡略モデル

    1. 入力 token を長さ V の One-hot にする。
    2. 埋め込み行列との積で、語彙より小さい次元 D の中間表現を得る。
    3. 別の線形変換と Softmax で、次の token に対する V 個の確率を出す。
    4. 正解 One-hot との Cross-entropy loss を求め、Backpropagation で重みを更新する。
    5. 繰り返し学習した後、中間層を token embedding として取り出す。

    講義の toy example は語彙6、埋め込み2次元だが、実際には語彙は数万〜数十万、埋め込みは数百〜数千次元になり得る。768は例として挙げられた値であり、固定規則ではない。大きい埋め込みは表現力を増やす一方、計算量・推論遅延・コストも増えるため、下流タスクとのトレードオフで選ぶ。

    残る弱点

    Word2vec の埋め込みは token ごとに固定で、語順も周囲の文脈も変えない。同じ綴りの bank が「銀行」と「川岸」で異なる意味を持っても、同じベクトルになる。この弱点が文脈を順番に取り込む RNN への動機になる。

    6. RNN と LSTM：語順を得て、長距離で詰まる

    単語ベクトルを単純平均すると語順が失われる。RNN（Recurrent Neural Network）は、時刻 t の token と直前の hidden state を受け取り、新しい hidden state を作る。hidden state は「ここまで読んだ系列」の要約であり、同じ計算セルを時刻方向へ繰り返す。

    タスクごとの使い方

    • 分類：系列末尾の hidden state をラベル空間へ射影する。
    • Token classification：対象 token の state をラベルへ射影する。
    • 生成：入力系列を読み終えた context / hidden state を Decoder の初期情報として使う。

    長距離依存と勾配

    情報を1本の state に順次圧縮すると、長い系列の初期情報を保ちにくい。学習では最終時刻の誤差を時間方向へ戻す BPTT（Backpropagation Through Time）を行うが、多数の微分を掛け続けるため、値が1未満なら勾配が0へ近づく vanishing gradient、1より大きければ増大する exploding gradient が起こり得る。

    LSTM（Long Short-Term Memory）は hidden state に加えて cell state を持ち、何を保持・忘却・出力するかを gate で制御して、長期依存を学びやすくする。ただし逐次性は残る。時刻 t を計算するには前の state が必要なので、系列を一括並列化しにくく、長い入力では学習が遅い。

    方式得られたもの主要な弱点
    Word2vec語の意味的な静的埋め込み語順・文脈を持たない
    RNN語順と「ここまで」の文脈単一 state への圧縮、長距離依存、勾配、逐次計算
    LSTMcell state と gate による長期記憶の改善完全解決ではなく、逐次計算のボトルネックも残る

    7. Attention と Self-attention

    必要箇所へ直接つなぐ

    従来の Encoder–Decoder は、入力系列全体を固定長ベクトルへ圧縮しがちだった。翻訳中の各出力 token が、入力のどこに対応するかを直接参照できれば、遠い情報を state の連鎖だけで運ばなくてよい。Bahdanau ら（2014）の Attention は、出力を予測するたびに関連する Encoder state を重み付けして取り出す。

    Self-attention：系列内を相互参照する

    Transformer は再帰を外し、各 token の表現を同じ系列内のすべての token から直接作る。これが Self-attention である。例えば teddy bear の表現を、文中の cute、is、reading などとの関係から更新する。したがって bank も、周囲が川か金融かによって別の context-aware embedding になる。

    Query・Key・Value

    Query（Q）

    「いま更新したい token は、どの情報を必要としているか」という問い合わせ。

    Key（K）

    各 token が Query とどれだけ対応するかを照合するための索引。

    Value（V）

    照合後の重みに従って実際に集約する内容ベクトル。

    Q・K・V は人が固定する属性ではない。同じ入力埋め込みを、学習可能な行列 WQ、WK、WV へ射影して得る。Q と K の類似度が「どこを見るか」を決め、その重みで V を加重平均する。全 token 分を行列演算として表せるため、GPU が得意な並列計算へ落とし込める。

    8. Transformer アーキテクチャ

    元論文の用途は機械翻訳で、左の Encoder が原言語を文脈化し、右の Decoder が訳文を1 token ずつ生成する。両側に Attention と token 単位の FFN（Feed-Forward Network）を積み重ねる。

    場所Attention の役割Q / K / V の由来
    Encoder Self-attention入力 token を、入力全体の文脈で更新すべて Encoder 側
    Decoder Masked Self-attention生成済み token だけを参照し、未来を見ないすべて Decoder 側、未来位置を mask
    Cross-attention現在の出力状態から、原文の必要箇所を参照Q は Decoder、K/V は Encoder 出力

    構成要素

    • Positional Encoding：Self-attention 自体は並び順を持たないため、token embedding に位置ベクトルを加える。元論文は sin / cos を使用する。
    • Multi-head Attention：異なる WQ/WK/WV を持つ複数 head で Attention を並列実行し、異なる関係を学べる自由度を与える。各 head を連結し、WO で元のモデル次元へ戻す。
    • FFN：各位置へ同じ2層ネットワークを適用する。講義では、内部次元を入力より大きくし、特徴を豊かにする自由度を与えると説明する。
    • Residual connection / Layer normalization：構成図の Add & Norm に含まれる。講義では詳細を掘り下げないが、元論文では各 sub-layer の周囲に置かれる。
    • Label smoothing：正解を確率1に固定せず、少量を他クラスへ配る。自然言語では次語の妥当解が複数あり得るため、過度な確信を抑え、元論文では BLEU 改善に寄与した。

    9. Scaled Dot-Product Attention を分解する

    入力行列を X とすると、まず Q = XWQ、K = XWK、V = XWV を作る。講義後半は、この行列計算を1段ずつ読み解く。

    Attention(Q, K, V) = softmax(QKT / √dk)V Vaswani et al. (2017), Equation (1)

    全 Query × Key のスコア"] K["Key 行列 K"] --> DOT DOT --> SCALE["√dₖ で割る
    スコアを安定化"] SCALE --> S["行ごとに Softmax
    Attention weight"] S --> SUM["重み × V
    Value の加重和"] V["Value 行列 V"] --> SUM SUM --> Z["文脈化された表現"]

    図をクリックすると拡大・ドラッグ・ズームできます。

    1. QKT：各 Query と全 Key の内積を取り、token 間の対応スコア行列を作る。
    2. √dk で除算：次元が大きいほど内積の絶対値が大きくなり、Softmax が極端になりやすい。スケーリングで安定させる。
    3. Softmax：各 Query の行を、全 Key に対する重みの分布へ変える。
    4. ×V：重みに従って Value を足し合わせ、各 Query に対応する context-aware vector を得る。
    5. Multi-head：この計算を異なる射影で h 回並列に行い、結果を列方向に連結して WO で射影する。

    Q と K は内積を取るため同じ次元が必要で、講義では dQ = dK と説明する。各 head が異なる関係を学ぶことを保証する明示的制約はないが、目的関数を改善する過程で異なる表現へ分化することが多い。

    10. 翻訳を端から端まで追う

    最後に A cute teddy bear is reading. を原文として、元論文型 Transformer の推論を通しで確認する。

    1. Tokenize：原文を token へ分割し、BOS（Beginning of Sequence）と EOS（End of Sequence）を付ける。
    2. 入力表現：各 token embedding に、sin / cos による positional encoding を要素ごとに加える。
    3. Encoder：入力から Q/K/V を作り、Multi-head Self-attention と FFN を通す。この block を N 回積み、文脈化された原文表現を得る。
    4. Decoder 開始：最初は BOS だけを入れる。Masked Self-attention は現在位置より右を見ないため、推論時と学習時の因果性を守る。
    5. Cross-attention：Decoder の状態を Q、Encoder の最終出力を K/V とし、今の出力に必要な原文箇所を集約する。
    6. 次 token：Decoder block と FFN を重ね、Linear + Softmax で語彙全体の確率分布を得て次 token を選ぶ。
    7. 自己回帰：選んだ token を Decoder 入力へ戻し、同じ処理を繰り返す。EOS が生成されたら停止する。
    この講義の到達点

    token 化、埋め込み、位置、Self-attention、Multi-head、Encoder、因果 Decoder、Cross-attention、FFN、語彙 Softmax が1本につながった。以降の LLM を理解する際も、「何を Q とし、どこから K/V を取るか」「未来を mask するか」「どの表現を並列に更新するか」が基本の読み方になる。

    11. 公式・一次資料で補う専門用語集

    以下は講義内の説明を核に、公式コース資料、原論文、ACL Anthology、Stanford の教科書資料で定義と注意点を補ったもの。

    NLP

    自然言語を分析・理解・生成する計算処理の総称。分類、系列ラベリング、生成など、入出力形式の異なるタスクを含む。

    出典: Jurafsky & Martin, Speech and Language Processing

    Token / Tokenization

    モデルが扱う離散的なテキスト単位と、その単位列へ分割する処理。token は単語・subword・文字と一致するとは限らない。

    出典: CME295 Lecture 1 slides

    OOV / Subword / BPE

    OOV は語彙にない入力。BPE 系 subword は未知語を既知の部分へ分解し、有限語彙でも希少語を構成しやすくする。

    出典: Sennrich et al. (2016), ACL Anthology

    Embedding

    離散 token を、学習可能な低次元の密ベクトルへ写した表現。類似する使用文脈がベクトル空間の幾何へ反映される。

    出典: Mikolov et al. (2013)

    Word2vec / CBOW / Skip-gram

    CBOW は周辺から中央語、Skip-gram は中央語から周辺を予測する。予測課題を通じて分散表現を学ぶ。

    出典: Efficient Estimation of Word Representations

    Proxy task

    本当に欲しい表現を直接教師付けせず、関連する予測課題を解かせて有用な内部表現を学習させる設計。

    出典: Mikolov et al. (2013), Distributed Representations

    RNN / BPTT

    直前の state を再利用して系列を順番に処理するネットワーク。BPTT は展開した時系列に沿って誤差を逆伝播する。

    出典: CME295 Lecture 1 slides

    Vanishing / Exploding Gradient

    多段の微分積で勾配が0へ消える、または極端に増える現象。長期依存の学習を難しくする。

    出典: Bengio, Simard & Frasconi (1994)

    LSTM

    cell state と gate を導入し、長期にわたる誤差信号と情報保持を改善する recurrent architecture。

    出典: Hochreiter & Schmidhuber (1997)

    Attention

    各出力時点で、入力側のどの state が重要かを重み付けして参照する。固定長ベクトルだけに圧縮する bottleneck を緩和する。

    出典: Bahdanau, Cho & Bengio (2014)

    Self-attention / QKV

    同じ系列から Q・K・V を作り、各 token を系列全体の加重和で文脈化する。Q/K は対応度、V は集約内容を担う。

    出典: Vaswani et al. (2017)

    Scaled Dot-Product / Multi-head

    QKᵀ を √dk で縮尺し Softmax 後に V を集約する。複数 head は異なる射影空間でこの計算を並列化する。

    出典: NeurIPS 2017 proceedings

    Positional Encoding

    再帰を持たない Transformer へ token の順序を与える信号。元論文は異なる周波数の sin / cos を embedding に加算する。

    出典: Vaswani et al. (2017), §3.5

    Causal Mask / Cross-attention

    Causal mask は Decoder が未来 token を参照するのを防ぐ。Cross-attention は Decoder の Q で Encoder の K/V を読む。

    出典: Vaswani et al. (2017), §3.2.3

    BLEU / ROUGE

    BLEU は機械翻訳、ROUGE は主に要約の自動評価として提案された参照ベース指標。単一の人間評価そのものではない。

    BLEU / ROUGE

    Perplexity

    系列の平均負対数尤度を指数化した値。正解 token へ高い確率を付けるほど低くなるが、生成品質の全側面は測れない。

    出典: Jurafsky & Martin

    Label Smoothing

    正解ラベルの確率質量を少し他クラスへ配り、過度に確信した分類器になるのを抑える。Transformer 元論文は ε=0.1。

    出典: Vaswani et al. (2017), §5.4

    BOS / EOS

    系列の開始・終了を表す special token。Decoder は BOS から生成を始め、EOS を選ぶと停止する。

    出典: CME295 Lecture 1 slides
