    1. イベント概要と 3 つの戦略テーマ

    Microsoft Build 2026 は 2026 年 6 月 2 日、サンフランシスコで開催された。Satya Nadella の基調講演に続き、Microsoft AI CEO の Mustafa Suleyman が新モデル群を、Windows + Devices 担当 EVP の Pavan Davuluri が Windows プラットフォームを、Applied Sciences Group の Steven Bathiche が Project Solara を発表。NVIDIA の Jensen Huang、Qualcomm の Cristiano Amon、Mayo Clinic CEO の Gianrico Farrugia なども登壇した。

    Microsoft が今回のメッセージの軸にしたのは、開発者向けに公開された GitHub の Kyle Daigle 名義の公式ブログで明示された 3 つのテーマ である。

    ① Intelligence that's truly yours

    Microsoft Agent Platform × Microsoft IQ で、コンテキスト・ガバナンス・モデル選択をトレードオフなく統合した「自社のものになる知能」を提供する。

    ② The full stack, your way

    シリコン → OS → 開発ツール → クラウドまで、開発者が好きなツール・モデル・ワークフローで構築できる選択肢を、Windows を中心に拡張する。

    ③ The future belongs to builders

    エージェント基盤を、コードを超えて科学・量子・新しい端末形態にまで広げる。Microsoft Discovery GA、Majorana 2、Project Solara が代表例。

    Jay Parikh のブログで提示された、エンタープライズ向けエージェントを「単一の統合システム」として運用するための 6 ステップ。各層で発表されたプロダクトが連結する全体像になっている。

    3. Microsoft IQ ― エージェント向けの文脈レイヤー

    Microsoft IQ は、エージェントを「世界の知識」と「組織の知識」の両方に接地（grounding）させる新しいコンテキストレイヤー。Build 2026 で GitHub Copilot／Microsoft Foundry／Copilot Studio で一般提供（GA） された。

    IQ ファミリー役割状態出典
    Work IQ Microsoft 365、組織システム、外部ソース上の「実際の働き方」（人・メール・文書・会議とそのつながり）を捉える職場知能レイヤー。Work IQ API は 6/16 に GA 予定。 GA / API 6月16日 GA aka.ms/WorkIQAPI_GA
    Fabric IQ 構造化ビジネスデータ（Data Agents、Ontology、Semantic Models）に共有セマンティック基盤を提供。 Preview aka.ms/azure-data-Build26
    Foundry IQ 企業知識と Web を横断する検索プランニング層。Knowledge Bases は GA、Serverless（Developer tier）は public preview、Foundry IQ MCP server で MCP 互換ホスト（Claude／ChatGPT／LangChain／Microsoft Agent Framework 等）から利用可能。 KB GA / Serverless Preview aka.ms/FoundryIQ
    Web IQ（新） エージェント向けの「AI ネイティブな Web 検索」。Bing インデックスをエージェント実行ループ向けに再設計した MCP 対応 API。P95 sub-165 ms、competitor 比 約 2.5 倍高速、ゼロデータリテンション、publisher preferences を尊重。 Limited Access aka.ms/nextgengrounding（aka.ms/WebIQ）

    Foundry IQ の主要更新

    • Foundry IQ Serverless（Developer tier）：Compute Unit ベースの従量課金、アイドル時にゼロスケール。課金開始は 2026 年後半予定。
    • 新しい knowledge sources（preview）：Work IQ、Fabric IQ、File Search、Azure SQL、MCP Server をマルチソース KB で統合。
    • Agentic retrieval 品質改善：回答品質ベンチマークを最大 20% 改善、single-shot RAG 比 recall を最大 54% 向上。
    • Security 更新：cross-tenant CMK（federated identity）、Purview sensitivity-label auditing、incremental SharePoint permissions sync、APIM 統合、Shared Private Link / NSP は GA。
    • Data pipeline：SharePoint ASPX / Lists の first-class インデックス、Azure Content Understanding chunking + image verbalization（preview）。

    4. MAI モデル ― Microsoft AI 自社開発 7 モデル

    Mustafa Suleyman 率いる Microsoft AI Superintelligence Team が、社内開発の 7 モデルからなる MAI ファミリー を発表。すべて ゼロ蒸留（zero distillation）／クリーンかつ商用ライセンス可能なデータ でゼロから学習。Foundry／GitHub Copilot／VS Code／PowerPoint／OneDrive に展開され、OpenRouter／Fireworks／Baseten 経由でも提供される。

    モデル用途特徴
    MAI-Thinking-1 推論／チャット／コーディング 35B アクティブパラメータの mid-size 推論モデル、256K コンテキスト。Blind test で Sonnet 4.6 を上回り、SWE Bench Pro で Opus 4.6 とコーディングが互角。Foundry で private preview。
    MAI-Code-1-Flash エージェント型コーディング 5B active parameters、Haiku 並み性能でより安価。GitHub Copilot／VS Code に深く統合。
    MAI-Image-2.5（+ Flash） 画像生成・編集 text-to-image で Arena AI 3 位、image-to-image で 2 位（Nano Banana 2 を超え）。PowerPoint で稼働、OneDrive に展開中、Foundry で提供。
    MAI-Transcribe-1.5 音声書き起こし 43 言語で SOTA 精度、競合比 5 倍高速。speaker diarization と domain-specific terminology biasing 対応。ストリーミング対応近日。
    MAI-Voice-2（+ Flash 近日） テキスト読み上げ 15 言語の自然な音声生成、短いサンプルから音声をクローン可能。新しい voice options 追加。

    Microsoft AI は同時に、目標として 「Humanist Superintelligence（人間中心の超知能）」 を掲げ、独自シリコン Maia 200 と組み合わせて 1.4 倍の効率向上 を達成済みであることも公表した。

    出典：Building a hill-climbing machine: Launching seven new MAI models（Microsoft AI） ／ aka.ms/MAI-Build

    Mayo Clinic との医療フロンティアモデル共同開発

    Microsoft と Mayo Clinic は、Mayo Clinic の臨床知見・de-identified 臨床データ・長期インサイトと、Microsoft の基盤 AI を組み合わせた 医療向けフロンティアモデル を共同開発すると発表。まず Mayo Clinic 環境内で診断・治療計画を支援し、検証後に Azure Foundry 経由で他組織にも提供。モデルの所有権は Mayo Clinic 側に残る。

    5. Microsoft Foundry の Build 2026 アップデート

    Microsoft Foundry は、本番エージェントに必要な「ランタイム・ツール・メモリ・知識・モデル・観測性・統治」のすべてを揃えるプラットフォーム。Build 2026 では特に 本番エージェントのホスティング、ツール接続、メモリ、評価／統治 が前進した。

    5-1. 開発者ツールと SDK

    • Microsoft Agent Framework：エージェントハーネス（skills／memory／middleware）、GitHub Copilot SDK／Claude Agent SDK 連携、Magentic-One ほかマルチエージェント・オーケストレーション・パターンが Stable。File system tools／memory tools／deep research agent は Public Preview。
    • Foundry Toolkit for VS Code：テンプレートまたは GitHub Copilot からエージェントを生成、ローカル debug、Toolboxes 連携、Foundry Agent Service へ deploy。GA

    5-2. エージェント本番運用

    • Hosted Agents in Foundry Agent Service：セッションごとの sandbox、専用 compute／memory／filesystem。2026 年 7 月初頭に GA 予定。Responses API（OpenAI 互換）と Invocations protocol（schema-free）両対応。OpenClaw／Hermes 等の long-running 自律エージェントもサポート。Routines（タイマー／スケジュール）も preview。
    • Toolboxes in Foundry：あらゆるツール種別を 1 つの managed エンドポイントに集約、認証・ライフサイクル・統治を Foundry が引き受け。Public Preview
    • Skills（preview）：プロジェクトスコープのカタログで version 管理、MCP リソースとして発見可能。Tool search も preview。
    • Voice Live：音声認識・TTS・割り込み処理・アバター等を 1 API に統合。Prompt agents 向けは GA、Hosted Agents + Voice Live は Public Preview。
    • Memory in Foundry Agent Service（preview）：Procedural Memory（新／Tau-bench で +7〜14% 絶対成功率向上）／User Memory／Session Memory の 3 種類。
    • Microsoft Teams / Microsoft 365 Copilot への publish：あらゆる Foundry エージェントを Teams や M365 Copilot に直接配置可能。2026 年 6 月に GA 予定。

    5-3. モデルとコンピュート

    • 新 MAI モデル群：Thinking-1／Image-2.5／Transcribe-2／Voice-2 が Foundry catalog で public preview に。
    • Fireworks AI on Foundry：オープンモデル推論を単一 Azure エンドポイントから提供。GA、PTU Data Zone 対応、SOC 2 ready。
    • Managed Compute in Foundry Models：リージョン GPU 制約を回避し、グローバルにワークロードをルーティング。
    • Fine-tuning / Frontier Tuning：GPT-5.5 比で 10 倍以上コスト効率（技術ドキュメント生成タスク）。Frontier Tuning は private preview。

    5-4. 信頼・観測性・セキュリティ

    • ASSERT：ポリシーを実行可能な評価に変換する OSS フレームワーク。LangChain／CrewAI／LightLLM／OpenAI 等に対応。aka.ms/assert
    • Agent Control Specification（ACS）：input／LLM／state／tool execution／output の 5 チェックポイントで decisive な制御を定義するオープン業界仕様。YAML で portable。Infosys／KPMG／IBM／Aviatrix／BigSpin／CrewAI 等が参画。
    • Guided Guardrail Setup：質問形式で PII フィルタ／jailbreak 保護／task adherence などを自動推奨。Public Preview
    • Rubric evaluator：エージェント定義から評価基準を自動生成し、重み付き集約スコアを Agent Optimizer に供給。
    • Tracing & Evaluations for any framework：LangChain／Semantic Kernel／カスタム framework のエージェントにも Foundry の production-grade tracing を適用。
    • Agent Optimizer in Foundry Agent Service：Foundry の評価スイートを AI Operations Service 内で実行し、本番シグナルから ranked な改善案にループ。近日 public preview。
    • ROI for agents in Foundry（private preview）：task completion rate、time saved、cost efficiency を計測。
    • Codename MDASH：100+ エージェントを deploy する multi-model agentic security system。data flow／business logic／exploit chains を推論して exploitable bugs を発見、Defender Portal で context-aware fixes を提供。

    出典：What's new in Microsoft Foundry | Build Edition ／ aka.ms/BuildFoundryAgents ／ aka.ms/FoundryIQ

    6. GitHub Copilot ― エージェントネイティブ開発体験

    GitHub の Chief Product Officer Mario Rodriguez が公開したブログで、エージェント時代の開発体験を全面再設計した GitHub Copilot app と関連アップデートが発表された。Microsoft 全体の中でも、最も実装が進んでいる「Build（構築）」レイヤーの主役にあたる。

    6-1. GitHub Copilot app（technical preview）

    • エージェントネイティブなデスクトップ体験。Pro／Pro+／Business／Enterprise ユーザーが対象。
    • My Work ビューで、接続済みリポジトリ横断のセッション・issue・PR・background automations を一元管理。
    • 各セッションは独立した git worktree で隔離実行。手動セットアップやブランチ管理不要。
    • Agent Merge：CI／reviewer／checks を監視し、設定した条件まで PR を自動運用。Copilot Max（Pro／Pro+／EDU からのアップグレード）で大量利用に対応。

    6-2. Canvas ― 意図を検査可能な作業面に

    チャット履歴ではなく、計画／PR／ブラウザセッション／ターミナル／ダッシュボード等を 双方向に編集できる canvas を提供。エージェントは canvas を更新し、開発者はその場で承認・並び替え・redirect できる（agent experience = AX）。

    6-3. Cloud and Local Sandboxes for GitHub Copilot

    • Local sandboxes：ファイルシステム・ネットワーク・システム機能のアクセスを制限した分離環境で Copilot を実行。中央でポリシー設定可。
    • Cloud sandboxes：GitHub がホストする完全分離・ephemeral な Linux 環境。任意のデバイスから session を継続できる remote control 対応。

    6-4. Copilot code review の拡張

    • Medium tier review：高推論モデルで PR をレビューし精度・recall を向上。リポジトリ別に low / medium を設定可能。
    • カスタム agent skills、MCP server connection、Actions workflows でレビュー基準を組織標準に拡張可。
    • /security-review skill、/rubberduck skill（GA、複数モデルで実装を批評）。
    • Azure DevOps でも Copilot code review がネイティブに利用可能に。

    6-5. ひとつのランタイム：SDK・CLI・cloud agent

    • GitHub Copilot SDK：Node.js／TypeScript／Python／Go／.NET／Rust／Java で GA。Copilot app と同じ agentic runtime を利用。
    • Copilot CLI：TUI を刷新、voice input（on-device speech-to-text、音声はマシンから出ない）、/every による定期タスク。
    • Copilot cloud agent：schedule／GitHub events トリガで issue 起票やコメント返信。デフォルトで write 前に確認、信頼後に autopilot へ。
    • Memory++ と /chronicle：アプリ・CLI・VS Code・GitHub にまたがるセッション履歴の継続参照。
    • Partner agent apps：LaunchDarkly、Bright、Amplitude、Sonar、Endor Labs、Octopus Deploy、Packfiles、PagerDuty、Miro 等。

    6-6. ハイブリッド compute と GitHub Copilot CLI /fleet

    GitHub Copilot CLI の /fleet で、クラウド上の primary agent が計画を立て、タスクの複雑度を評価し、ローカルモデルで処理可能なものを自動で local subagent に委譲。Surface RTX Spark Dev Box などローカル compute を活かしてコスト削減。

    出典：GitHub Copilot app: The agent-native desktop experience（GitHub Blog）

    7. Windows ― エージェントを開発・実行する最も信頼されるプラットフォーム

    Pavan Davuluri が発表した Windows プラットフォームのアップデートは、「Windows for developers, period.」（Windows 開発者のための Windows ではなく、すべての開発者のための Windows）を掲げ、開発者体験、エージェント実行基盤、オンデバイス AI の 3 つを大幅に強化。

    7-1. 開発者最適化された Windows 11 体験

    項目内容状態
    Coreutils for WindowsRust 製の GNU Coreutils 互換 CLI 群（uutils ベース）。Linux／macOS／WSL／コンテナ／クラウド間でコマンドを統一。GA
    WSL containersWindows 上で Linux コンテナを built-in で実行する CLI／API。IT 管理者向けの policy-based 制御。Public Preview 近日
    Windows Developer Configurationsdev-config.winget 1 行で VS Code／GitHub Copilot／WSL／PowerShell 7／Git／GitHub CLI／Python と開発者向け設定を一括導入。GA
    Intelligent TerminalACP（Agent Communication Protocol）で agent CLI を Windows Terminal に統合。失敗コマンドの修正提案を agent pane に表示。Experimental
    Windows Development SkillsWinUI3 skills + winapp CLI でエージェントにネイティブ Windows app 構築の知識を付与。GA
    Windows 365 with Developer configurationクラウド PC として、同じ開発者向け構成を提供。Public Preview

    7-2. エージェントを安全に実行する OS 基盤

    • Microsoft Execution Containers（MXC）：policy-driven な実行レイヤー。開発者がファイル／ネットワーク等のアクセスを宣言、Windows が runtime で強制。GitHub Copilot CLI が fast process isolation を採用済み。Process／session isolation は Windows Insider に近日提供。github.com/microsoft/mxc
    • Agent 365 と MXC のネイティブ統合：Defender／Entra／Intune／Purview 保護を 2026 年 7 月に preview で適用、ローカルエージェントを企業ガバナンス配下に。
    • OpenClaw on Windows：node／gateway を MXC で contained 実行、Windows companion app から claws を管理。
    • NVIDIA OpenShell on Windows：MXC ベースの autonomous agent runtime に policy management／inference routing／PII obfuscation を追加。
    • Windows 365 for Agents：Intune-managed Cloud PC で computer-using agent を実行。GA
    • Hermes Agent / Manus / OpenAI Codex が MXC を採用または評価中。

    7-3. オンデバイス AI ― Unmetered Intelligence on Windows

    • Aion 1.0 Instruct：Windows 既存 SLM より小型・高速・効率的。summarization／rewrite／intents／accessibility 用途。Edge Insider channels で preview 配布、7 月に Hugging Face で open weights 公開予定。
    • Aion 1.0 Plan：14B パラメータの reasoning + tool-calling モデル（32K コンテキスト）。capable device に in-box 提供、フルローカルなエージェントワークフローを実現。
    • Speech Recognition API：on-device の real-time / batch speech-to-text。NPU／CPU で実行可能。最初は英語のみ、順次拡大。
    • Windows AI APIs を CPU／GPU に拡大：既存 inbox SLM（Phi Silica）が capable GPU で動作、Video Super Resolution / Speech Recognition が CPU 対応に。Public Preview
    • Microsoft Foundry on Windows：ローカル AI を Foundry 経由で組み込む経路を提供。

    7-4. Windows のセキュリティ強化（デフォルトで risk を下げる）

    • Post-Quantum Cryptography（PQC）：Windows TLS の PQ hybrid key exchange、CNG API での composite PQC アルゴリズム、ADCS による PQ 証明書発行。
    • IAKerb と LocalKDC を Windows Insider Preview の Server / Client で設定可能に。NTLM 利用削減。
    • Windows Hardware Compatibility Program（WHCP）認証ドライバをデフォルト化する staged transition。
    • Smart App Control と App Control for Business の reputation-based enforcement を拡張。

    出典：Build 2026: Furthering Windows as the trusted platform for development（Windows Blog）

    8-2. DGX Station for Windows

    NVIDIA GB300 Grace Blackwell Ultra Superchip を Windows プラットフォームに統合した、deskside の AI スーパーコンピュータ。最大 1 兆パラメータのフロンティアモデルをローカル実行可能、常時起動の frontier AI エージェントを企業内アプリ／ワークフローに接続できる。2026 年 Q4 提供予定。

    8-3. Project Solara ― エージェント特化デバイス向けプラットフォーム

    Steven Bathiche（Applied Sciences Group）が発表した、「アプリ中心」から「エージェント中心」へと computing を再設計する chip-to-cloud プラットフォーム。just-in-time UI（適応的に生成される UI）、企業向け manageability／identity／security を組み込み、複数エージェントを共存させる open multi-agent 世界を前提とする。

    • OS 基盤：Microsoft Device Ecosystem Platform（MDEP）（AOSP ベースの enterprise OS）、Intune、Entra ID、Hello for Business、物理マイクミュート等。
    • 2 つのリファレンス端末コンセプト：
      • Badge concept（携帯型／Qualcomm wearable シリコン）：タッチスクリーン、指紋認証、far-field マイクアレイ、サイドカメラ、5G／WiFi／Bluetooth／GNSS。胸元バッジの形態で常時エージェントを携帯。
      • Desk concept（据置型／MediaTek IoT シリコン）：タッチスクリーン、顔認証、UWB プレゼンスセンサー、デュアル far-field マイク。USB-C 接続で Windows 365 Client へ変身可。
    • パイロット参加企業：AccuWeather、Best Buy、CVS Health、Levi's、Target ほか。
    • シリコンパートナー：Qualcomm（wearable）／MediaTek（stationary）。
    • Microsoft 内では Microsoft 365 Copilot（voice）／Researcher／Facilitator／Priority Agent、外部では GitHub Copilot や Dragon Copilot が活用を検討。

    出典：Composing a new platform for agent-first devices（Command Line） ／ aka.ms/ProjectSolara

    9. Azure Cobalt 200 ― 自社 Arm CPU の第 2 世代

    Azure 自社設計の cloud-native CPU の第 2 世代 Azure Cobalt 200 Arm-based VMs が早期アクセスプレビューで提供開始。Cobalt 100 比で 50% の世代間性能向上を実現し、agentic AI ワークロードに最適化。

    9-1. Cobalt 200 SoC の主要スペック

    • Arm Neoverse V3 ベース、TSMC 3nm（N3P）プロセス、chiplet アーキテクチャ。
    • 1 コアあたり 3 MB の L2 cache、システム全体 192 MB の L3 cache。
    • カスタムアクセラレータ（圧縮・暗号）、カスタムメモリコントローラ、メモリ暗号化をデフォルト有効化。
    • 最大 128 vCPU、Azure Boost 統合により remote storage IOPS +20%／throughput +10%、NW 帯域 +15%。

    10. データ・データベース・分析プラットフォーム

    10-1. Azure HorizonDB（public preview）

    新しい エンタープライズ対応 Postgres 互換 DB サービス。Microsoft Entra ID 統合、プライベートエンドポイント、保存／通信暗号化、Postgres 拡張エコシステムへの完全互換に加え、AI 時代の機能をネイティブ統合。

    • 最大 128 TB の DB、最大 15 read replica、サブミリ秒の AZ 跨ぎコミットレイテンシ。比較対象の自己管理セットアップに対し 3 倍以上のスループット。
    • DiskANN + spherical quantization による高速・省メモリの vector search を Postgres に統合。
    • Hybrid search：pgvector + pg_textsearch 拡張で semantic + keyword 検索を 1 クエリに。
    • AI モデル管理：GPT モデル／ランキングモデルを DB と並走で登録・バージョン管理。
    • AI Functions（azure_ai 拡張）：SQL／アプリロジックから直接モデル推論を呼び出し。
    • AI Pipelines：データ変更をトリガに event-driven なモデル実行・データ処理。VS Code 拡張で可視化。
    • 提供リージョン（preview 開始）：Australia East、Central US、Sweden Central、West US 2、West US 3。今後 East US、Canada Central、Indonesia Central、Italy North、Japan East、Korea Central、Poland Central。

    出典：Azure HorizonDB: Enterprise-Ready Postgres, Engineered for the AI Era

    10-2. Rayfin ― AI ファーストなアプリケーションバックエンド

    Microsoft Fabric 上で動作する open-source SDK + CLI。データモデル／API／アクセスポリシー／ビジネスロジックをコードで定義し、Fabric へ deploy するだけで enterprise-grade のアプリバックエンドが構築される。GitHub Copilot などのコーディングエージェントが scaffold・拡張できるよう strongly typed。Replit と統合し、Replit でビルドしたアプリを Fabric tenant に deploy する経路を提供。Leatherman などが先行採用。

    出典：Introducing Rayfin（Microsoft Fabric Community Blog） ／ aka.ms/rayfin

    15. まとめ ― 「単一のエージェントシステム」を成立させる年

    Microsoft Build 2026 の発表を俯瞰すると、過去数年で散発的に投入してきた要素（Foundry、Agent 365、Copilot、IQ、Windows、Surface、Azure シリコン、量子）が、「エンタープライズエージェントを構築・運用するためのひとつの統合システム」として整列した年だったといえる。

    • 「自社のものになる知能」として、Microsoft IQ（Work／Fabric／Foundry／Web）と Frontier Tuning が、企業のデータと業務を学習できる仕組みを提供。
    • 「フルスタックを好きなように」として、Windows（MXC、Aion、Intelligent Terminal、Coreutils）と Foundry（Hosted Agents、Toolboxes、Voice Live、Memory）が、ローカル／クラウド／IDE／CLI のあらゆる流儀に対応。
    • 「次のフロンティア」として、Project Solara（端末）、Majorana 2（量子）、Microsoft Discovery（科学）、Mayo Clinic 連携（医療）、Maia 200 + GB200 / GB300（コンピュート）が、開発者プラットフォームを越えた未来軸を提示。

    セキュリティとガバナンスの面でも、ASSERT / ACS / MDASH など open standard と OSS を組み合わせ、エンタープライズ／開発者コミュニティ両方を巻き込む姿勢が強い。MAI モデルの「ゼロ蒸留」と Mayo Clinic との「モデル所有権は顧客側」など、責任ある AI に関する明示的なメッセージも一貫している。
